[Redux] Redux 原始碼解析

前言

最近仔細研究了一下Redux的原始碼發現其實蠻短的，在這邊紀錄一下。

createStore

createStore 用於產生 store，可以傳入reducer , 初始 state , middleware。

{ type: ActionTypes.INIT }是用於初始化 state 的 action ， 如此一來即使我們還沒發出任何 action 但目前的 state 可以得到預設值，也因此 reducer 中若無符合的 action 要回傳預設 state。

getState

單純返回目前的 state 物件

dispatch

subscribe

subscribe 會把事件放到nextListeners陣列，在 dispatch action 後會執行所有 subscribe 的事件，而執行 subscribe 後會回傳一個函式用於 unsubscribe。

這邊的 listeners 分別是上一個 dispatch 後的狀態以及接著註冊事件後的狀態。

下一個 dispatch 發出後才會更新 listeners 的狀態。

replaceReducer

更換目前的 reducer

observable

這部分似乎是為了 observable 所預留的接口，在這邊先跳過

combineReducers

combineReducers 的任務就是將許多 reducers 合併成一個 reducers ，查看原始碼可以發現大部分在做錯誤處理。

assertReducerShape這個錯誤處理比較奇怪，從錯誤訊息可以得知希望不要使用@@redux為頭的 Action Type，應該要直接回傳初始 state，但從判斷式中可以知道就算真的用了這個名稱也不會回傳 undefined 才對，而且就算真的回傳 undefined 也該在第一個地方擋下來了。

bindActionCreators用途是將多個 action 和 dispatch 產生一個綁定的物件，如此一來直接呼叫該物件的函式即直接 dispatch 該 action。
